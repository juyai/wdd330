The Trusty Alert

--show a dialog at certain points in the code.

--alert() stops a program from running until OK is clicked

Using the console

console.log() method can be used to log the value of variables at different stages of
the program, although it will not actually stop the execution of the program in the same way as alert() does.

The console.trace() method will log an interactive stack trace in the console.
This will show the functions that were called in the lead up to an exception occurring while the code is running.

Error Objects

When an exeception occurs an error object can be created by the host eviroment
it can also be created in the code using a constructor function.

const error = new Error();

This constructor function takes a parameter that's used as the error message:

const error = new Error('Oops, something went wrong');

Seven more error objects used for specific errors:

  • RangeError--when a number is outside an allowable range of values.
  • ReferenceError--when a reference is made to an item that doesn't exist. For example, calling a function that hasn't been defined.
  • SyntaxError--when there's an error in the code's syntax.
  • TypeError--when there's an error in the type of value used; for example, a string is used when a number is expected.
  • URIError--a problem encoding or decoding the URI.
  • InternalError--an error occurs in the JavaScript engine. A common cause of this too much recursion.

error objects can also be used as constructors to create custom error objects:
const error = new TypeError('You need to use numbers in this function');

Throwing Exceptions

any problems in your code to be highlighted and dealt with, rather than lurk quietly in the background.

throw statement can be applied to any JavaScript expression

cause the execution of the program to stop.

function squareRoot(number) {
'use strict';
if (number < 0) {
throw new RangeError('You can't find the square root of negative numbers')
}
return Math.sqrt(number);
};

Exception Handling

try, catch, and finally

If we suspect a piece of code will result in an exception, we can wrap it in a try block.

function imaginarySquareRoot(number) {
'use strict';
try {
return String(squareRoot(number));
} catch(error) {
return squareRoot(-number)+'i';
}
}

The code inside the catch block will only run if an exception is thrown inside the try block

A finally block can be added after a catch block

always be executed after the try or catch block, regardless of whether an exception occurred or not.

if you want some code to run in both cases.
